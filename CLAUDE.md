## Agent Interaction Rules

### Understand Before Acting

Before modifying files, running implementation commands, or changing code, first state a brief understanding of the user's request.

The understanding should cover the requested outcome, relevant constraints, and any ambiguity that may affect execution.

### Use A Directory-Style Overview For Lists

When responding with bullet points or numbered lists, start with a brief directory-style overview.

The overview should name the topics or sections covered by the list. It should not be a detailed summary or conclusion.

### Follow The Latest Requirement

During requirement, design, or plan discussions, treat the user's latest change as the current source of truth.

If the user changes, replaces, or removes an earlier point, stop relying on the outdated version unless explicitly asked.

Do not defend the old version or add compatibility logic only to preserve a superseded idea, unless correctness or the user explicitly requires it.
